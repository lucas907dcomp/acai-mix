## User Story

**Como** sistema,  
**Quero** sincronizar automaticamente as vendas offline quando a internet reconectar,  
**Para que** nenhuma venda se perca e o histórico no Supabase fique completo.

---

## Acceptance Criteria

- [ ] AC1: `syncStore` em `src/stores/syncStore.ts` com estado: `isOnline`, `pendingCount`, `isSyncing`
- [ ] AC2: `addPending(sale)` grava venda no Dexie com `synced: false`
- [ ] AC3: Listener `window.addEventListener('online', drain)` inicia sync ao reconectar
- [ ] AC4: `drain()` processa todas as `pending_sales` com `synced: false` em sequência
- [ ] AC5: Cada venda drenada faz UPSERT (não INSERT) no Supabase: `INSERT ... ON CONFLICT (id) DO UPDATE SET synced_at = now()`
- [ ] AC6: Após UPSERT bem-sucedido, atualiza `synced: true` no Dexie
- [ ] AC7: Se UPSERT falhar, incrementa `sync_attempts` e deixa na fila (tenta novamente no próximo `online` event)
- [ ] AC8: `pendingCount` reflete número de vendas não sincronizadas em tempo real

---

## Tasks

### T1 — syncStore Zustand (2h)
```typescript
interface SyncStore {
  isOnline: boolean
  pendingCount: number
  isSyncing: boolean
  addPending: (sale: Sale) => Promise<void>
  drain: () => Promise<void>
  initListeners: () => void
}
```

### T2 — Listener online/offline (0.5h)
- [ ] `initListeners()` chamado em `src/App.tsx` no mount
- [ ] `window.addEventListener('online', () => { setIsOnline(true); drain() })`
- [ ] `window.addEventListener('offline', () => setIsOnline(false))`
- [ ] `navigator.onLine` para estado inicial

### T3 — Drain com UPSERT (1.5h)
- [ ] Busca do Dexie: `db.pending_sales.where('synced').equals(0).toArray()`
- [ ] Para cada venda: `supabase.from('sales').upsert(sale, { onConflict: 'id' })`
- [ ] Sucesso: `db.pending_sales.update(sale.id, { synced: true })`
- [ ] Erro: `db.pending_sales.update(sale.id, { sync_attempts: sale.sync_attempts + 1, last_sync_error: err.message })`
- [ ] Atualizar `pendingCount` ao final

## Dev Notes

**UPSERT para idempotência:** UUID gerado no cliente garante que mesmo se a venda chegou ao Supabase antes (via retry), o UPSERT não cria duplicata.

**Ordem do drain:** Processar em ordem de `created_at ASC` — vendas mais antigas primeiro, mantendo consistência temporal dos totais do turno.

**pendingCount:** Calculado via `db.pending_sales.where('synced').equals(0).count()` — atualizar após cada operação de drain.

**initListeners:** Chamar apenas uma vez no mount da App. Usar useRef para garantir idempotência.
