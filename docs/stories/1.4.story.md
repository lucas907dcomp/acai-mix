## User Story

**Como** dono do projeto,  
**Quero** o frontend deployado em uma URL pública desde o início,  
**Para que** eu possa acompanhar o progresso e o dono possa testar em seu dispositivo real.

---

## Acceptance Criteria

- [ ] AC1: Projeto conectado ao Vercel via GitHub (deploy automático em push para `main`)
- [ ] AC2: Variáveis de ambiente `VITE_SUPABASE_URL` e `VITE_SUPABASE_ANON_KEY` configuradas no painel do Vercel
- [ ] AC3: URL de produção funcional e carregando a aplicação sem erros de console
- [ ] AC4: Preview deployments ativos para PRs (feature branches geram URL de preview)
- [ ] AC5: `npm run build` é o comando de build no Vercel (`dist` como output dir)

---

## Scope

**IN:**
- Deploy inicial no Vercel
- Configuração de env vars no Vercel
- Deploy automático em push para main

**OUT:**
- CI/CD completo com testes automatizados
- CDN customizado ou domínio próprio
- Configuração de staging environment separado
