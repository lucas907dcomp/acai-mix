## User Story

**Como** operador do caixa,  
**Quero** ser notificado quando meu turno for fechado automaticamente,  
**Para que** eu saiba que preciso abrir um novo turno para continuar vendendo.

---

## Acceptance Criteria

- [ ] AC1: Quando `shiftStore.activeShift` vai de `open` para `null` (detectado via polling), exibe toast informativo
- [ ] AC2: Toast: "Turno encerrado automaticamente. Inicie um novo turno para continuar."
- [ ] AC3: POSPage exibe `ShiftOpenScreen` imediatamente após detecção do fechamento
- [ ] AC4: Polling do turno ativo a cada 60s (detecta fechamento feito pela Edge Function)
- [ ] AC5: Sem notificação se o operador fechou manualmente (evitar duplo aviso)

---

## Tasks

### T1 — Polling de turno ativo (1h)
- [ ] Em `shiftStore`, adicionar `startPolling()` e `stopPolling()`
- [ ] Intervalo: 60s via `setInterval`

- [ ] Detectar transição `activeShift: Shift → null` (não disparar no boot quando já era null)
- [ ] Exibir toast com `sonner` ou Shadcn toast
- [ ] POSPage reage ao `activeShift === null` e renderiza `ShiftOpenScreen`

## Dev Notes

**Detecção de fechamento externo:**
```typescript
// Em shiftStore — comparar estado anterior
const previousShiftId = useRef(activeShift?.id)
useEffect(() => {
  if (previousShiftId.current && !activeShift) {
    toast.info('Turno encerrado automaticamente. Inicie um novo turno.')
  }
  previousShiftId.current = activeShift?.id
}, [activeShift])
```

**Alternativa:** Supabase Realtime subscription em `shifts` (mais elegante, mas adiciona complexidade de WebSocket). Para Fase 1, polling 60s é suficiente.
