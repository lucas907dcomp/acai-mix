## User Story

**Como** operador do caixa,  
**Quero** poder digitar o peso manualmente quando a balança está com defeito,  
**Para que** a loja não pare de vender por causa de uma falha de hardware.

---

## Acceptance Criteria

- [ ] AC1: `ManualInputProvider` implementado em `src/providers/scale/ManualInputProvider.ts`
- [ ] AC2: `ManualInputProvider` implementa `IScaleProvider` com `type: 'manual'`
- [ ] AC3: Expõe método `setWeight(grams: number)` para que a UI possa injetar o peso
- [ ] AC4: Ao chamar `setWeight()`, emite o valor via `onWeight` callback (mesmo contrato do SerialScaleProvider)
- [ ] AC5: `connect()` e `disconnect()` são no-ops (sem hardware para conectar)
- [ ] AC6: Componente `ManualWeightInput` em `src/components/pdv/ManualWeightInput.tsx` — input numérico touch-friendly que chama `setWeight()` ao confirmar

---

## Scope

**IN:**
- `ManualInputProvider` (lógica pura)
- Componente `ManualWeightInput` (UI de entrada)

**OUT:**
- Lógica de ativação/desativação do modo manual (fica no scaleStore — Story 3.4)
- Dialog de confirmação para ativar modo manual (Story 3.4)

---

## Tasks

### T1 — Implementar ManualInputProvider (0.5h)
- [ ] `type = 'manual'`
- [ ] `connect()` → `onConnectionChange(true)` imediato
- [ ] `disconnect()` → `onConnectionChange(false)`
- [ ] `setWeight(grams)` → chama callback registrado em `onWeight`

### T2 — Criar ManualWeightInput (1h)
- [ ] Input numérico grande (touch-friendly, font-size grande)
- [ ] Botão "Confirmar Peso" que chama `provider.setWeight(value)`
- [ ] Clear automático após confirmar
- [ ] Validação: peso > 0 e < 10000g (razoável para açaí)
- [ ] Placeholder: "Digite o peso em gramas"

### T3 — Testes unitários (0.5h)
- [ ] Testa que `setWeight()` dispara `onWeight` callback com o valor correto
- [ ] Testa validação de peso inválido
