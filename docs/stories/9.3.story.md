## Dev Notes

**A migration já foi escrita pelo @data-engineer (Dara) e está em:** `supabase/migrations/20260521000000_sale_cancellation.sql`

**Conteúdo da migration (resumo para o dev não precisar abrir o arquivo):**
1. ADD COLUMN `status TEXT NOT NULL DEFAULT 'COMPLETED' CHECK (IN ('COMPLETED','CANCELLED'))`
2. ADD COLUMN `cancelled_at TIMESTAMPTZ`
3. ADD COLUMN `cancelled_by UUID REFERENCES auth.users(id) ON DELETE SET NULL`
4. CHECK constraint: `COMPLETED → cancelled_at IS NULL`, `CANCELLED → cancelled_at IS NOT NULL`
5. Índices partial: `idx_sales_completed_location_created` e `idx_sales_cancelled_at`
6. Trigger: `trg_update_shift_totals_on_update` (AFTER UPDATE OF status, WHEN status muda)
7. Views recriadas: `shift_summary` e `daily_summary` com `FILTER (WHERE sl.status = 'COMPLETED')`
8. Função: `can_cancel_sale(p_sale_id UUID)` — SECURITY DEFINER, STABLE
9. Policy: `sales_update_cancel_admin` — FOR UPDATE, USING (is_admin() AND can_cancel_sale(id))

**Por que `types/index.ts` NÃO é gerado automaticamente:**  
`supabase gen types` gera `src/types/supabase.ts` (tipos raw do banco). O `src/types/index.ts` é uma camada de tipos customizados do projeto (interface `Sale`, `Shift`, etc.) que são mapeados manualmente para maior controle. A geração automática não sobrescreve esta camada — é uma decisão arquitetural do projeto.

**Backfill gratuito:**  
O `ALTER TABLE ... ADD COLUMN status TEXT NOT NULL DEFAULT 'COMPLETED'` backfill todas as linhas existentes com `status='COMPLETED'` de forma atômica — sem migration de dados separada, sem script de backfill.

**Gotcha de recursão RLS:**  
`can_cancel_sale` usa `SECURITY DEFINER` para evitar o erro 42P17 (recursão infinita em policies self-referential). Padrão idêntico ao `is_admin()` já validado em produção. **Não alterar** para `SECURITY INVOKER`.
