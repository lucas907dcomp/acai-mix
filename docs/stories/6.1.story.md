### T2 — Criar db.ts (1h)
```typescript
// src/lib/db.ts
import Dexie, { type Table } from 'dexie'
import type { Sale, Shift } from '@/types'

export interface PendingSale extends Sale {
  synced: boolean
  sync_attempts: number
  last_sync_error?: string
}

export interface ProvisionalShift extends Omit<Shift, 'id'> {
  local_id: string
  remote_id?: string
  status: 'provisional' | 'synced'
}

class AcaiMixDB extends Dexie {
  pending_sales!: Table<PendingSale>
  provisional_shifts!: Table<ProvisionalShift>

  constructor() {
    super('acaimix-db')
    this.version(1).stores({
      pending_sales: 'id, shift_id, synced, created_at',
      provisional_shifts: 'local_id, status, location_id'
    })
  }
}

export const db = new AcaiMixDB()
```

## Dev Notes

**Dexie versioning:** Sempre incrementar version ao alterar schema — não modificar version existente em produção.

**IndexedDB limits:** Chrome permite até ~60% do disco disponível. Para PDV com ~100 vendas/dia, sem problema.

**Singleton:** `db` é exportado como instância única do módulo — Vite garante que módulos são singletons por padrão.
