## User Story

**Como** sistema,  
**Quero** manter o estado do turno ativo em memória com Zustand,  
**Para que** todos os componentes do PDV acessem o shift_id sem re-fetch desnecessário.

---

## Acceptance Criteria

- [ ] AC1: `shiftStore` em `src/stores/shiftStore.ts` com estado: `activeShift`, `isLoading`, `error`
- [ ] AC2: `openShift()` faz POST no Supabase e popula `activeShift`
- [ ] AC3: `closeShift()` faz PATCH `status: 'closed'` e limpa `activeShift`
- [ ] AC4: `loadActiveShift()` busca turno aberto no Supabase no boot da app
- [ ] AC5: `activeShift.id` disponível globalmente para `saleStore.confirmSale()` como `shift_id`
- [ ] AC6: Persiste `activeShift.id` no sessionStorage para sobreviver a refresh sem novo fetch

---

## Tasks

### T1 — Criar shiftStore (1.5h)
```typescript
interface ShiftStore {
  activeShift: Shift | null
  isLoading: boolean
  error: string | null
  loadActiveShift: (locationId: string) => Promise<void>
  openShift: (locationId: string) => Promise<void>
  closeShift: (closedBy: string) => Promise<void>
  updateTotals: (sales: number, amount: number) => void
}
```

### T2 — loadActiveShift no boot (0.5h)
- [ ] Chamar em `src/App.tsx` após autenticação confirmada
- [ ] Busca: `shifts.select('*').eq('status', 'open').eq('location_id', locationId).maybeSingle()`
- [ ] Se encontrado: setar `activeShift`
- [ ] Se não: `activeShift = null` (POSPage mostrará ShiftOpenScreen)

### T3 — Persistência sessionStorage (0.5h)
- [ ] `persist` middleware do Zustand com `sessionStorage`
- [ ] Persistir apenas `activeShift.id` (não o objeto completo — reidrata no boot)
