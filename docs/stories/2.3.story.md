## User Story

**Como** sistema AçaiMix,  
**Quero** que rotas protegidas verifiquem autenticação e role antes de renderizar,  
**Para que** um Staff não consiga acessar o dashboard financeiro e usuários não autenticados sejam redirecionados para login.

---

## Acceptance Criteria

- [x] AC1: Componente `ProtectedRoute` implementado em `src/router/ProtectedRoute.tsx`
- [x] AC2: Usuário não autenticado tentando acessar qualquer rota protegida → redirect para `/login`
- [x] AC3: Staff tentando acessar `/dashboard` ou `/admin/settings` → redirect para `/pos`
- [x] AC4: Admin acessa todas as rotas sem restrição
- [x] AC5: Loading spinner exibido enquanto o perfil do usuário está sendo carregado (evita flash de redirect)
- [x] AC6: Rotas configuradas: `/pos` (staff + admin), `/dashboard` (admin only), `/admin/settings` (admin only)

---

## Scope

**IN:**
- Componente `ProtectedRoute` com verificação de role
- Configuração de todas as rotas protegidas no router
- Redirect para login e redirect por role
- Loading state durante verificação de auth

**OUT:**
- Lógica de negócio de cada página (outras stories)
- Proteção baseada em permissões granulares (roles simples na Fase 1)
