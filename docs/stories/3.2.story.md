## User Story

**Como** operador do caixa,  
**Quero** que o peso da balança UPX Wind D3 apareça automaticamente na tela,  
**Para que** eu não precise digitar o peso manualmente a cada venda.

---

## Acceptance Criteria

- [ ] AC1: `SerialScaleProvider` implementado em `src/providers/scale/SerialScaleProvider.ts` implementando `IScaleProvider`
- [ ] AC2: `connect()` abre diálogo Web Serial API para o usuário selecionar a porta da balança
- [ ] AC3: Parser correto para o protocolo serial da UPX Wind D3 (baud rate e formato de pacote documentados no Dev Notes)
- [ ] AC4: Peso é emitido em gramas (número decimal) via `onWeight` callback
- [ ] AC5: Reconexão automática tentada se a porta desconectar inesperadamente (máximo 3 tentativas)
- [ ] AC6: `onConnectionChange` emite `false` quando porta desconectar, `true` quando reconectar
- [ ] AC7: Funciona apenas em Chrome/Edge desktop — detecta suporte e lança erro descritivo se não disponível

## Dev Notes

**Protocolo a ser documentado após pesquisa:** [Source: docs/architecture/acaipro-system-design.md#6.3]

**Protocolo hipotético (a validar na instalação):**
```
Formato esperado: STX + peso (6 dígitos ASCII) + unidade + CR + LF
Exemplo: \x02 000500 g \r\n → 500 gramas
Baud rate provável: 9600 ou 4800
Data bits: 8, Stop bits: 1, Parity: None
```

**IMPORTANTE:** Na ausência do manual, implementar com o protocolo hipotético acima. O `IScaleProvider` abstraction permite ajustar o parser no dia da instalação sem afetar o resto do sistema.

**Web Serial API — Chrome only:** Confirmar durante testes que funciona apenas em Chrome/Edge 89+. [Source: docs/architecture/acaipro-system-design.md#14]
