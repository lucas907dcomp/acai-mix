## User Story

**Como** usuário do AçaiMix (Admin ou Staff),  
**Quero** fazer login com email e senha,  
**Para que** eu acesse apenas as funcionalidades permitidas para o meu perfil.

---

## Acceptance Criteria

- [x] AC1: Página `/login` com campos email, senha e botão "Entrar"
- [x] AC2: Login via `supabase.auth.signInWithPassword()` com tratamento de erro (credenciais inválidas)
- [x] AC3: Após login bem-sucedido, busca perfil do usuário em `user_profiles` para obter `role` e `location_id`
- [x] AC4: Redirect automático baseado em role: Admin → `/dashboard`, Staff → `/pos`
- [x] AC5: Loading state visível durante a requisição
- [x] AC6: Mensagem de erro clara para credenciais inválidas (sem expor detalhes técnicos)
- [x] AC7: Sessão persiste entre recarregamentos de página (Supabase gerencia refresh token automaticamente)
- [x] AC8: Botão "Esqueci minha senha" visível (pode redirecionar para fluxo Supabase padrão — não bloqueia AC)

## Dev Notes

**Fluxo de auth:** [Source: docs/architecture/acaipro-system-design.md#10.1]
1. Supabase Auth: email + senha → JWT
2. Frontend busca `user_profiles` para role + location_id
3. Zustand persiste perfil na sessão
4. React Router verifica role em rota protegida

**authStore mínimo:**
```typescript
interface AuthStore {
  user: User | null
  profile: UserProfile | null
  isLoading: boolean
  setAuth: (user: User, profile: UserProfile) => void
  clearAuth: () => void
}
```

**Nota de segurança:** Client-side route protection é UX, não segurança. A segurança real é no RLS do banco. [Source: docs/architecture/acaipro-system-design.md#10.2]
