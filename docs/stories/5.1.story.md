## User Story

**Como** operador do caixa,  
**Quero** abrir meu turno ao chegar ao trabalho com um clique,  
**Para que** todas as vendas sejam associadas corretamente ao meu turno.

---

## Acceptance Criteria

- [ ] AC1: Página `/pos` exibe tela de "Iniciar Turno" quando não há turno ativo para a location
- [ ] AC2: Botão "Iniciar Turno" cria registro em `shifts` com `status: 'open'`, `opened_at: now()`, `location_id` do operador
- [ ] AC3: Após abertura, `shiftStore.activeShift` é populado e PDV fica acessível
- [ ] AC4: Se já existe turno aberto para a location, PDV carrega diretamente (sem tela de abertura)
- [ ] AC5: `shift_number` é auto-incrementado por location (1, 2, 3...)
- [ ] AC6: Apenas usuários com role `staff` ou `admin` da location correta podem abrir turno

## Dev Notes

**Schema shifts:** [Source: docs/architecture/acaipro-system-design.md#4.1]
```sql
CREATE TABLE shifts (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  location_id UUID NOT NULL REFERENCES locations(id),
  shift_number INTEGER NOT NULL,
  opened_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  closed_at TIMESTAMPTZ,
  closed_by UUID REFERENCES auth.users(id),
  status TEXT NOT NULL DEFAULT 'open' CHECK (status IN ('open', 'closed')),
  total_sales INTEGER DEFAULT 0,
  total_amount NUMERIC(10,2) DEFAULT 0
);
```

**Verificação de turno ativo:** Feita no mount do POSPage — não usar localStorage (pode ficar stale entre sessões).
