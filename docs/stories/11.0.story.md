## User Story

**Como** desenvolvedor/@data-engineer do AçaiMix,
**Quero** um ambiente de teste local isolado (stack Supabase CLI + seed sintético), reproduzível a qualquer momento,
**Para que** eu possa aplicar e validar as migrations e o RLS do EPIC-11 sem tocar no único banco real existente hoje — o de produção, com as 2 lojas vendendo.

---

## Contexto (por que esta story existe)

Verificado durante a revisão da migration (@data-engineer): este projeto **não tem staging**. `.env`/`.env.example` apontam para um único projeto Supabase, e `package.json` usa `supabase gen types --linked` (projeto linkado = produção). O único banco disponível hoje é o de produção. O que já existe, mas nunca foi documentado como processo, é `supabase/config.toml` — a stack local do Supabase CLI, que sobe um Postgres/Auth isolado, aplica todas as migrations do zero e roda um seed. Esta story formaliza esse processo.

## Dev Notes

**Esta story é sobre infraestrutura de teste, não sobre a lógica do EPIC-11.** A migration `20260714000000_owner_multiloja.sql` já existe e será aplicada automaticamente ao rodar `supabase start` (porque o Supabase CLI aplica todas as migrations do diretório em ordem). Isso significa que, tecnicamente, o schema do EPIC-11 já vai existir no ambiente local assim que esta story terminar — mas a **validação** de que esse schema se comporta como esperado (RLS correta, guarda-corpo funcionando) é trabalho das Stories 11.1 e 11.2, não desta.

**`auth.users` não aceita INSERT direto com senha usável.** O `seed.sql` de produção já documenta isso ("criar via Supabase Dashboard, depois inserir em user_profiles"). Para o ambiente local, usar a CLI (`supabase` tem comandos de auth admin) ou a API REST local do GoTrue (disponível em `http://localhost:54321` por padrão) para criar os usuários de teste com senha conhecida.

**Nunca rodar isto contra o projeto de produção.** A connection string usada em T1/T3 deve ser sempre a local (porta padrão 54322 para Postgres direto, ou via `supabase db` que já aponta para o ambiente certo quando não há `--linked`).
