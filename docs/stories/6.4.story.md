## User Story

**Como** sistema,  
**Quero** que o backend valide e reconcilie vendas sincronizadas offline,  
**Para que** vendas associadas ao turno errado (por virada de turno durante offline) sejam corrigidas automaticamente.

---

## Acceptance Criteria

- [ ] AC1: Edge Function `sync-sales` aceita POST com array de vendas a sincronizar
- [ ] AC2: Para cada venda: valida `created_at` contra limites do `shift_id` informado
- [ ] AC3: Se `created_at` cai fora do shift informado → reatribui ao turno correto com base no timestamp
- [ ] AC4: Campo `sync_reconciled: true` marcado nas vendas reatribuídas
- [ ] AC5: UPSERT no banco: `INSERT INTO sales ... ON CONFLICT (id) DO UPDATE SET synced_at = now(), sync_reconciled = ...`
- [ ] AC6: Retorno: `{ synced: number, reconciled: number, errors: Array<{id, error}> }`
- [ ] AC7: Falha parcial não aborta o batch — processa o que der, reporta erros

### T2 — Lógica de reconciliação (2h)
```typescript
async function reconcileSale(sale: PendingSale): Promise<ReconciledSale> {
  const shift = await getShift(sale.shift_id)
  
  if (isWithinShiftWindow(sale.created_at, shift)) {
    return { ...sale, sync_reconciled: false }
  }
  
  // Buscar turno correto pelo timestamp
  const correctShift = await findShiftByTimestamp(sale.created_at, sale.location_id)
  return { ...sale, shift_id: correctShift.id, sync_reconciled: true }
}
```

## Dev Notes

**Segurança:** Edge Function valida JWT automaticamente via Supabase. RLS garante que operador só acessa dados da própria location.

**findShiftByTimestamp:**
```sql
SELECT * FROM shifts 
WHERE location_id = $1 
  AND opened_at <= $2 
  AND (closed_at IS NULL OR closed_at > $2)
LIMIT 1
```

**Batch size:** Máximo 50 vendas por request para evitar timeout da Edge Function (default: 26s Supabase).

**Falha parcial:** Processar cada venda individualmente num try/catch. Não usar transação única que aborta tudo.
