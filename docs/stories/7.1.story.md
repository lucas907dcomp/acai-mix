## Tasks

### T1 — Migration com views (2h)
```sql
-- shift_summary view
CREATE OR REPLACE VIEW shift_summary AS
SELECT 
  s.id,
  s.location_id,
  s.shift_number,
  s.opened_at,
  s.closed_at,
  s.status,
  COUNT(sl.id) AS total_sales,
  COALESCE(SUM(sl.amount), 0) AS total_amount,
  CASE WHEN COUNT(sl.id) > 0 
    THEN ROUND(SUM(sl.amount) / COUNT(sl.id), 2) 
    ELSE 0 END AS avg_ticket,
  EXTRACT(EPOCH FROM (COALESCE(s.closed_at, now()) - s.opened_at)) / 60 AS duration_minutes
FROM shifts s
LEFT JOIN sales sl ON sl.shift_id = s.id
GROUP BY s.id;

-- daily_summary view
CREATE OR REPLACE VIEW daily_summary AS
SELECT 
  DATE(sl.created_at AT TIME ZONE 'America/Sao_Paulo') AS sale_date,
  sl.location_id,
  COUNT(*) AS total_sales,
  SUM(sl.amount) AS total_amount,
  COUNT(DISTINCT sl.shift_id) AS total_shifts
FROM sales sl
GROUP BY sale_date, sl.location_id;
```

## Dev Notes

**security_invoker:** Views no PostgreSQL executam com permissões do criador por padrão. Usar `WITH (security_invoker = true)` para que RLS do chamador seja aplicado.

**Timezone:** Dashboard exibe dados em `America/Sao_Paulo`. Views convertem `created_at` (UTC) para timezone local.

**daily_summary:** Agrupa por data local (Brasília), não UTC — uma venda às 23:30 de segunda em Brasília não aparece como terça.
