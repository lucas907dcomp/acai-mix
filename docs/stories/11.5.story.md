## User Story

**Como** dono das 2 açaiterias,
**Quero** um usuário `owner` já vinculado às minhas 2 lojas reais,
**Para que** eu possa logar uma única vez e ver os números das duas lojas, sem que os 2 usuários `admin` que já uso hoje sejam afetados de nenhuma forma.

## Dev Notes

**Esta story toca produção.** É a primeira do epic que sai do ambiente de teste (Story 11.0) para o banco real. Antes de T3, confirmar explicitamente com o usuário: (a) o email a ser convidado como owner, (b) os 2 `location_id` reais das lojas (consultar `SELECT id, name FROM locations` em produção, nunca assumir).

**`auth.users` não aceita INSERT direto com senha usável** — mesma observação já registrada em `seed.sql`: criar o usuário via Supabase Dashboard, depois vincular via SQL.

**Idempotência:** o `ON CONFLICT DO NOTHING` em `user_locations` (chave primária composta `user_id, location_id`) evita duplicar vínculo se o script for rodado 2x por engano. O `INSERT` em `user_profiles` não deve usar `ON CONFLICT` sem cuidado — se o usuário já existir, um `UPDATE` acidental de `role`/`location_id` seria bloqueado pelo trigger de guarda da Story 11.1 de qualquer forma (defesa em profundidade).
