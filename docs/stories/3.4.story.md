## User Story

**Como** desenvolvedor do AçaiMix,  
**Quero** um hook `useScale` e `scaleStore` que gerenciem o ciclo de vida da balança,  
**Para que** qualquer componente possa ler o peso atual e o status de conexão sem se preocupar com a implementação.

---

## Acceptance Criteria

- [ ] AC1: `scaleStore` Zustand em `src/stores/scaleStore.ts` com estado: `isConnected`, `currentWeightGrams`, `provider`, `providerType`
- [ ] AC2: Hook `useScale.ts` em `src/hooks/useScale.ts` que conecta o provider ao store
- [ ] AC3: Ao montar o componente PDV, balança inicia com `MockScaleProvider` em desenvolvimento e `SerialScaleProvider` em produção (controlado por variável de ambiente `VITE_SCALE_MOCK=true`)
- [ ] AC4: Função `switchToManual()` no store troca o provider para `ManualInputProvider` após confirmação do usuário
- [ ] AC5: Quando `switchToManual()` é chamado, exibe dialog de confirmação antes de trocar
- [ ] AC6: `disconnect()` limpa subscriptions e reseta o estado quando componente desmonta (cleanup)

## Tasks

### T1 — Criar scaleStore (1h)
```typescript
interface ScaleStore {
  isConnected: boolean
  currentWeightGrams: number | null
  provider: IScaleProvider | null
  providerType: 'serial' | 'mock' | 'manual'
  setProvider: (provider: IScaleProvider) => void
  updateWeight: (grams: number) => void
  updateConnection: (connected: boolean) => void
  switchToManual: () => void
  disconnect: () => void
}
```

### T2 — Implementar useScale hook (1.5h)
- [ ] Instancia provider correto baseado em `VITE_SCALE_MOCK`
- [ ] Registra callbacks `onWeight` e `onConnectionChange` → atualiza store
- [ ] Cleanup no unmount: `provider.disconnect()`
- [ ] Tratamento de erro no `connect()` com mensagem amigável

### T3 — Dialog de modo manual (1h)
- [ ] Shadcn `Dialog` com mensagem de confirmação
- [ ] "A balança não está conectada ou você quer digitar manualmente. Esta venda será marcada como peso manual."
- [ ] Botões: "Confirmar Modo Manual" e "Cancelar"
- [ ] Ao confirmar: `setProvider(new ManualInputProvider())`

## Dev Notes

**scaleStore spec:** [Source: docs/architecture/acaipro-system-design.md#5.2]

**Variável de ambiente:**
```
VITE_SCALE_MOCK=true → MockScaleProvider
VITE_SCALE_MOCK=false (ou ausente) → SerialScaleProvider
```

**Importante:** Zustand permite que apenas o componente de display de peso re-renderize quando `currentWeightGrams` muda, sem afetar outros componentes do PDV. Este é o motivo de usar Zustand em vez de Context API. [Source: docs/architecture/acaipro-system-design.md#1.2]
