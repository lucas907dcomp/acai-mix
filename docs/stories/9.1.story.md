## User Story

**Como** administrador ou atendente da loja,  
**Quero** baixar um arquivo CSV com as vendas do período selecionado no Dashboard,  
**Para que** eu possa levar os dados para planilhas próprias, conciliação bancária e para o contador sem precisar de suporte técnico.

---

## Acceptance Criteria

- [ ] AC1: Botão "Exportar CSV" visível no `Dashboard.tsx`, ao lado dos seletores de período, para usuários com `role: 'admin'` e `role: 'staff'`
- [ ] AC2: O CSV gerado contém **todas** as vendas filtradas pelo mesmo período e location do dashboard — incluindo canceladas, com coluna `Status` marcada
- [ ] AC3: Colunas na ordem exata: `Data/Hora;Peso (g);Preço/g (R$);Valor (R$);Pagamento;Status` (cabeçalho em pt-BR)
- [ ] AC4: Encoding **UTF-8 com BOM** (`﻿`) para abrir corretamente no Excel pt-BR sem mojibake
- [ ] AC5: Separador `;` (ponto e vírgula) — vírgula é separador decimal no pt-BR
- [ ] AC6: `Data/Hora` em formato ISO 8601 com offset local (ex: `2026-05-20T14:32:00-03:00`)
- [ ] AC7: Coluna `Pagamento` com valores: `PIX`, `CARTAO`, `DINHEIRO` (cartão = credit + debit ambos mapeiam para `CARTAO`)
- [ ] AC8: Coluna `Status` com valores: `ATIVA` ou `CANCELADA`
- [ ] AC9: Nome do arquivo: `acaipro-vendas-{YYYY-MM-DD}-a-{YYYY-MM-DD}.csv`
- [ ] AC10: Geração **100% client-side** via `Blob + URL.createObjectURL` — sem nova chamada ao Supabase
- [ ] AC11: Se não houver vendas no período: botão desabilitado com tooltip "Sem vendas no período"
- [ ] AC12: Geração de até 5.000 linhas em < 500ms no navegador

## Dev Notes

**Período do Dashboard → datas para query:**  
O `period` state do Dashboard (`'today'`, `'week'`, `'month'`) precisa ser convertido em `from`/`to` para a query. Padrão a usar:
- `today`: `startOfDay(new Date())` até `endOfDay(new Date())`
- `week`: `subDays(new Date(), 6)` até `endOfDay(new Date())`
- `month`: `subDays(new Date(), 29)` até `endOfDay(new Date())`

`date-fns` já está no projeto (usado em `Dashboard.tsx` linha 2).

**Query com status incluído (após migration 9.3):**  
Após a Story 9.3 aplicar a migration, a tabela `sales` terá coluna `status`. A query desta story deve buscar todos os registros **sem** filtro de status para incluir canceladas no CSV (AC2). Interface `Sale` em `src/types/index.ts` será atualizada na Story 9.3.

**Nota de dependência de tipo:** Esta story pode ser implementada antes da 9.3 — o campo `status` na interface `Sale` é adicionado pela 9.3, mas a query `select('*')` já vai retornar o campo após a migration ser aplicada. Para desenvolver antes da 9.3, tipar `status` como `string | undefined` e fazer coerce no CSV. Se depois da 9.3, usar `sale.status as 'COMPLETED' | 'CANCELLED'` diretamente.

**Formato decimal pt-BR:**  
```typescript
// Usar toFixed + replace para evitar Intl que inclui símbolo R$
const formatDecimal = (n: number, places: number) =>
  n.toFixed(places).replace('.', ',')
```

**Reutilização de `period` state:**  
`Dashboard.tsx` já tem `const [period, setPeriod] = useState<DatePeriod>('today')`. O hook `useSalesForExport` deve receber `locationId` e `period` (ou `from`/`to` derivados) como parâmetros. Não duplicar o estado.

**locationId:** Disponível via `useAuthStore((s) => s.profile?.location_id)` — padrão do projeto.
