## User Story

**Como** dono da loja,  
**Quero** acessar o dashboard em qualquer dispositivo (PC, tablet, celular) com layout adaptado,  
**Para que** eu possa acompanhar o negócio de casa ou pelo celular.

---

## Acceptance Criteria

- [ ] AC1: `DashboardPage` em `src/pages/DashboardPage.tsx` acessível em `/dashboard`
- [ ] AC2: Rota protegida: apenas `role: 'admin'` tem acesso (staff é redirecionado para `/pos`)
- [ ] AC3: Layout desktop (≥1024px): sidebar + grid 2 colunas (cards + gráfico)
- [ ] AC4: Layout tablet (768px–1023px): sidebar colapsada + 2 colunas
- [ ] AC5: Layout mobile (<768px): bottom nav + stack vertical (1 coluna)
- [ ] AC6: Seletor de período no topo: Hoje / Esta Semana / Este Mês
- [ ] AC7: Indicador de última atualização: "Atualizado às HH:mm" com botão de refresh manual

## Dev Notes

**Layout responsivo — estratégia:**
- Desktop: sidebar fixa (260px) + conteúdo com grid
- Mobile: sidebar vira bottom navigation (5 ícones)
- Headless: sidebar já existe no AppLayout (Story 2.4) — DashboardPage apenas define o conteúdo

**Proteção admin:** ProtectedRoute existente (Story 2.3) aceita `role` como prop. Usar pattern já estabelecido.

**Acesso remoto (dono de casa):** Dashboard é uma página web normal — funciona em qualquer browser/dispositivo com login válido. Sem configuração especial para acesso remoto.
