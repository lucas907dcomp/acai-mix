## User Story

**Como** desenvolvedor do AçaiMix,  
**Quero** um projeto Supabase configurado com Auth, banco de dados e Edge Functions,  
**Para que** o backend esteja pronto para receber o schema e as funções da aplicação.

---

## Acceptance Criteria

- [ ] AC1: Projeto criado no dashboard do Supabase com nome "acaipro" (região: São Paulo se disponível, senão us-east-1)
- [ ] AC2: Supabase CLI instalado e projeto linkado localmente (`supabase link --project-ref <id>`)
- [ ] AC3: `supabase/config.toml` configurado com timezone `America/Sao_Paulo`
- [ ] AC4: Variáveis de ambiente configuradas: `VITE_SUPABASE_URL` e `VITE_SUPABASE_ANON_KEY` em `.env.local`
- [ ] AC5: `.env.local` adicionado ao `.gitignore`
- [ ] AC6: `supabase/functions/` criado para receber as Edge Functions futuras
- [ ] AC7: `supabase start` funciona localmente para desenvolvimento offline
- [ ] AC8: Cliente Supabase configurado em `src/lib/supabase.ts`

## Dev Notes

**Referência arquitetural:** [Source: docs/architecture/acaipro-system-design.md#2.2]

```typescript
// src/lib/supabase.ts
import { createClient } from '@supabase/supabase-js'

const supabaseUrl = import.meta.env.VITE_SUPABASE_URL
const supabaseAnonKey = import.meta.env.VITE_SUPABASE_ANON_KEY

export const supabase = createClient(supabaseUrl, supabaseAnonKey)
```

**Timezone crítico:** `America/Sao_Paulo` para Edge Functions de fechamento de turno às 16h e 23h. [Source: docs/architecture/acaipro-system-design.md#7.1]
