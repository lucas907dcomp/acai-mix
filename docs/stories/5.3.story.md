## User Story

**Como** sistema,  
**Quero** fechar os turnos abertos automaticamente às 16h e às 23h (horário de Brasília),  
**Para que** os totais do turno sejam calculados sem intervenção manual.

---

## Acceptance Criteria

- [ ] AC1: Edge Function `close-shift` deployada no Supabase com schedule `0 16,23 * * *` (timezone `America/Sao_Paulo`)
- [ ] AC2: Busca todos os turnos com `status: 'open'` e `opened_at < now()` (qualquer location)
- [ ] AC3: Calcula totais: `COUNT(sales)` e `SUM(sales.amount)` agrupados por `shift_id`
- [ ] AC4: Atualiza turno: `status: 'closed'`, `closed_at: now()`, `total_sales`, `total_amount`
- [ ] AC5: Cria próximo turno automaticamente com `status: 'open'` para cada location que tinha turno fechado
- [ ] AC6: Logs de execução visíveis no Supabase Dashboard (Edge Function logs)
- [ ] AC7: Idempotente: se chamada duas vezes, não fecha turno já fechado

## Dev Notes

**Cron schedule:** `0 16,23 * * *` = toda vez que hora bater 16:00 ou 23:00 UTC-equivalente
**Timezone:** Configurar `TZ=America/Sao_Paulo` na Edge Function (Deno suporta)

**Deno.cron (Supabase):**
```typescript
// supabase/functions/close-shift/index.ts
Deno.serve(async (req) => {
  // Invoke manual para testes
  return await runCloseShift()
})

Deno.cron('close-shift', '0 16,23 * * *', async () => {
  await runCloseShift()
})
```

**Conflito offline:** Esta Edge Function fecha o turno na nuvem. Vendas offline feitas após o fechamento serão reconciliadas pela Story 6.4. [Source: docs/architecture/acaipro-system-design.md#8.2]

**Auto-abertura:** Ao fechar o turno, abre o próximo automaticamente para não bloquear o PDV quando o operador voltar.
