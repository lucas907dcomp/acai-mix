# QA — Story 9.3: Migration Gate (Sale Cancellation Schema)

## Pré-requisitos

1. Migration aplicada via `supabase db push`
2. Types regenerados via `npm run db:types`

---

## Cenários SQL de Validação

Execute no Supabase SQL Editor ou via `psql`.

### 1. Verificar colunas adicionadas

```sql
SELECT column_name, data_type, is_nullable, column_default
FROM information_schema.columns
WHERE table_name = 'sales'
  AND column_name IN ('status', 'cancelled_at', 'cancelled_by')
ORDER BY column_name;
```

**Esperado:**
| column_name   | data_type                   | is_nullable | column_default |
|---------------|-----------------------------|-------------|----------------|
| cancelled_at  | timestamp with time zone    | YES         | null           |
| cancelled_by  | uuid                        | YES         | null           |
| status        | text                        | NO          | 'COMPLETED'    |

---

### 2. Verificar constraint de status

```sql
-- Deve falhar com CHECK constraint violation
INSERT INTO sales (location_id, shift_id, weight_grams, weight_source, price_per_gram, amount, payment_method, status)
VALUES ('00000000-0000-0000-0000-000000000001', '00000000-0000-0000-0000-000000000001', 300, 'scale', 0.055, 16.5, 'pix', 'INVALID_STATUS');
```

**Esperado:** `ERROR: new row for relation "sales" violates check constraint`

---

### 3. Constraint de consistência COMPLETED → cancelled_at IS NULL

```sql
-- Deve falhar: status COMPLETED não pode ter cancelled_at preenchido
INSERT INTO sales (location_id, shift_id, weight_grams, weight_source, price_per_gram, amount, payment_method, status, cancelled_at)
VALUES ('00000000-0000-0000-0000-000000000001', '00000000-0000-0000-0000-000000000001', 300, 'scale', 0.055, 16.5, 'pix', 'COMPLETED', NOW());
```

**Esperado:** `ERROR: new row for relation "sales" violates check constraint`

---

### 4. Constraint de consistência CANCELLED → cancelled_at IS NOT NULL

```sql
-- Deve falhar: status CANCELLED requer cancelled_at
INSERT INTO sales (location_id, shift_id, weight_grams, weight_source, price_per_gram, amount, payment_method, status)
VALUES ('00000000-0000-0000-0000-000000000001', '00000000-0000-0000-0000-000000000001', 300, 'scale', 0.055, 16.5, 'pix', 'CANCELLED');
```

**Esperado:** `ERROR: new row for relation "sales" violates check constraint`

---

### 5. INSERT sem status usa DEFAULT

```sql
INSERT INTO sales (location_id, shift_id, weight_grams, weight_source, price_per_gram, amount, payment_method)
VALUES ('00000000-0000-0000-0000-000000000001', '00000000-0000-0000-0000-000000000001', 300, 'scale', 0.055, 16.5, 'pix')
RETURNING id, status, cancelled_at;
```

**Esperado:** `status = 'COMPLETED'`, `cancelled_at = NULL`

---

### 6. Verificar views filtram CANCELLED

```sql
-- shift_summary e daily_summary só devem contar vendas COMPLETED
SELECT * FROM shift_summary WHERE shift_id = '<id_de_turno_com_venda_cancelada>';
```

**Esperado:** totais não incluem vendas com `status = 'CANCELLED'`

---

### 7. Verificar índices criados

```sql
SELECT indexname, indexdef
FROM pg_indexes
WHERE tablename = 'sales'
  AND indexname IN ('idx_sales_completed_location_created', 'idx_sales_cancelled_at');
```

**Esperado:** ambos os índices presentes.

---

### 8. Verificar função can_cancel_sale

```sql
-- Deve retornar false para sale já cancelada
SELECT can_cancel_sale('<id_de_venda_cancelada>');
```

**Esperado:** `false`

```sql
-- Deve retornar true para sale ativa
SELECT can_cancel_sale('<id_de_venda_ativa>');
```

**Esperado:** `true`

---

### 9. Trigger de reversão de totais (validar após 9.4)

Após implementação da story 9.4 (cancel online), validar:

```sql
-- 1. Obter totais antes do cancelamento
SELECT total_sales, total_amount FROM shift_summary WHERE shift_id = '<shift_id>';

-- 2. Cancelar a venda via função/RPC

-- 3. Conferir que totais diminuíram corretamente
SELECT total_sales, total_amount FROM shift_summary WHERE shift_id = '<shift_id>';
```

**Esperado:** `total_sales` decrementado em 1, `total_amount` decrementado pelo valor da venda cancelada, nunca negativo (GREATEST(..., 0)).
